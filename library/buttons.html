Button Library

Reference page for the current `.in-btn` API. This uses the live token files and compiled core stylesheet in this plugin so the examples reflect the real browser-facing contract: default, `sm`, `lg`, semantic variants, outline, block, and disabled states.

Base Sizes

The default button is the unsuffixed size token. Small and large are applied with modifiers.

Usage Notes

Use `.in-btn` as the base class. Add one optional size modifier (`.in-btn--sm` or `.in-btn--lg`), one optional semantic variant (`.in-btn--primary`, `.in-btn--secondary`, and so on), and then any structural modifier such as `.in-btn--outline` or `.in-btn--block`.
